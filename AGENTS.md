# دستورالعمل‌های ویژه برای Jules (ایجنت هوش مصنوعی)

سلام Jules،

این فایل شامل دستورالعمل‌ها و نکات ویژه‌ای است که باید در طول توسعه پروژه "آلفا" به آنها توجه کنی. لطفاً قبل از انجام هر کاری، این موارد را به دقت مطالعه کن.

## معماری و ساختار پروژه

* **ماژولار بودن:** سعی کن تمام بخش‌های پروژه را به صورت ماژولار و با کمترین وابستگی به یکدیگر طراحی کنی. این کار به ما کمک می‌کند تا در آینده بتوانیم هر بخش را به راحتی توسعه داده و نگهداری کنیم.
* **پیروی از ساختار:** همیشه از ساختار دایرکتوری‌های تعریف شده در `README.md` پیروی کن. کدهای مربوط به هر بخش باید در دایرکتوری مخصوص به خود قرار گیرند.

## هوش مصنوعی (AI)

* **الگوریتم‌های پیشرفته:** در پیاده‌سازی ماژول هوش مصنوعی، از الگوریتم‌های به‌روز و کارآمد استفاده کن. هدف ما ایجاد یک سیستم هوشمند واقعی است، نه یک سیستم مبتنی بر قوانین ساده.
* **ادغام با گیت‌هاب:** ماژول هوش مصنوعی باید به طور کامل با GitHub Actions یکپارچه شود و بتواند به صورت خودکار بر روی pull requestها و issueها اجرا شود.

## گردش کار (Workflow)

* **تست‌های خودکار:** برای تمامی بخش‌های پروژه، تست‌های خودکار بنویس. گردش کارهای GitHub Actions باید به گونه‌ای تنظیم شوند که این تست‌ها را به صورت خودکار اجرا کرده و از سلامت کدها اطمینان حاصل کنند.
* **بسته‌بندی و انتشار:** فرآیندهای کامپایل، بسته‌بندی و انتشار برنامه‌ها برای پلتفرم‌های مختلف باید به صورت خودکار و از طریق GitHub Actions انجام شوند.

## مستندات

* **مستندات جامع:** برای تمامی کدها و الگوریتم‌هایی که پیاده‌سازی می‌کنی، مستندات کامل و واضحی بنویس. این مستندات باید به گونه‌ای باشند که سایر توسعه‌دهندگان بتوانند به راحتی کدها را درک کرده و با آنها کار کنند.

## تعامل با کاربر

* **گزارش پیشرفت:** به طور منظم، گزارش پیشرفت کار خود را به کاربر ارائه بده.
* **درخواست بازخورد:** در صورت نیاز، از کاربر برای تصمیم‌گیری در مورد مسائل مهم و چالش‌برانگیز، درخواست بازخورد کن.
