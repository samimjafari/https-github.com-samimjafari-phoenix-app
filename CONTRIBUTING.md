# راهنمای مشارکت در پروژه آلفا

از اینکه به مشارکت در پروژه آلفا علاقه‌مند هستید، سپاسگزاریم! مشارکت شما به ما کمک می‌کند تا این سیستم را بهبود بخشیده و به اهداف آن نزدیک‌تر شویم.

## نحوه مشارکت

برای مشارکت در این پروژه، لطفاً مراحل زیر را دنبال کنید:

1. **یک Issue ایجاد کنید:** قبل از شروع هر کاری، لطفاً یک Issue جدید در مخزن پروژه ایجاد کرده و ایده‌ها، پیشنهادات یا باگ‌هایی که پیدا کرده‌اید را در آن مطرح کنید. این کار به ما کمک می‌کند تا از کارهای موازی جلوگیری کرده و در مورد بهترین راه‌حل با یکدیگر گفتگو کنیم.

2. **مخزن را Fork کنید:** پس از ایجاد Issue و توافق بر روی کلیات کار، مخزن پروژه را به حساب کاربری خودتان Fork کنید.

3. **یک شاخه (Branch) جدید بسازید:** برای هر ویژگی جدید یا رفع باگ، یک شاخه جدید با نامی توصیفی ایجاد کنید. به عنوان مثال:
   ```bash
   git checkout -b feature/new-cool-feature
   ```
   یا
   ```bash
   git checkout -b fix/annoying-bug
   ```

4. **تغییرات خود را اعمال کنید:** کدهای خود را نوشته و تغییرات لازم را اعمال کنید. لطفاً به موارد زیر توجه داشته باشید:
   - **استانداردهای کدنویسی:** سعی کنید از استانداردهای کدنویسی موجود در پروژه پیروی کنید.
   - **مستندات:** در صورت نیاز، مستندات مربوط به تغییرات خود را نیز به‌روزرسانی کنید.
   - **تست‌ها:** برای ویژگی‌های جدید، تست‌های مربوطه را نیز اضافه کنید.

5. **تغییرات خود را Commit کنید:** پس از اتمام کار، تغییرات خود را با یک پیام کامیت واضح و توصیفی، Commit کنید.

6. **یک Pull Request ارسال کنید:** در نهایت، یک Pull Request (PR) از شاخه خود به شاخه `main` مخزن اصلی ارسال کنید. در توضیحات PR، به Issue مربوطه اشاره کرده و توضیح دهید که تغییرات شما چه کاری انجام می‌دهند.

## گزارش باگ

اگر باگی در سیستم پیدا کردید، لطفاً یک Issue جدید با برچسب `bug` ایجاد کرده و اطلاعات زیر را در آن ارائه دهید:
- **توضیح باگ:** شرح کاملی از باگ و نحوه وقوع آن.
- **مراحل بازتولید:** مراحل دقیقی که برای بازتولید باگ باید طی شوند.
- **رفتار مورد انتظار:** آنچه که انتظار داشتید اتفاق بیفتد.
- **رفتار واقعی:** آنچه که در عمل اتفاق افتاده است.
- **اطلاعات سیستم:** سیستم‌عامل، نسخه مرورگر و سایر اطلاعات مرتبط.

از همکاری شما سپاسگزاریم!
